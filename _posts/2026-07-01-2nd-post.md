홈페이지 하나 만들려다 GitHub과 하루종일 씨름한 이야기


시작은 단순했다. "인터넷에 기록 남길 홈페이지 하나 있으면 좋겠다." 딱 그 정도 생각이었다.

1. 기획은 거창하게
처음엔 진지하게 기획서까지 썼다. 일상·여행, 투자·재테크, 업무·커리어 세 개 섹션으로 나누고, 공개 범위까지 표로 정리하고. 근데 정작 뭘 써야 할지는 하나도 안 정해졌다. 기획만 완벽했다.

결국 결론은 이거였다. 일단 아무거나 만들어보자. 기획은 나중에 다시 해도 된다.

2. GitHub 계정을 처음 만들다
GitHub 계정도 없었다. 구글 로그인으로 간단히 만들었다. kilimi84. 여기까진 순조로웠다.

kilimi84.github.io라는 이름으로 저장소를 만들고, HTML 파일 하나를 웹에서 드래그해서 올렸다. 신기하게도 몇 분 뒤 정말 kilimi84.github.io에 페이지가 떴다. 이 순간은 꽤 감격스러웠다.

3. 블로그 시스템을 붙이려다 폴더가 다 깨졌다
문제는 여기서부터였다. 글을 여러 개 쓰려면 Jekyll이라는 블로그 시스템이 필요하다고 해서 파일 여러 개를 준비했다. _posts, _layouts, assets 같은 폴더 구조가 있어야 하는데—

웹에서 드래그 앤 드롭으로 업로드하니까 폴더 구조가 전부 무시되고 파일들이 그냥 다 평평하게 올라갔다.

_layouts/default.html이어야 할 파일이 그냥 default.html로. Jekyll은 폴더 위치를 정확히 봐야 인식하는데, 이러니 사이트가 그냥 예전 화면 그대로 떴다. 대체 뭐가 문제인지 한참 헤맸다.

4. 파일 이름에 슬래시를 넣으면 폴더가 생긴다
해결법은 의외로 간단했다. GitHub 웹의 "Create new file" 화면에서, 파일 이름 칸에 _layouts/default.html처럼 슬래시를 직접 입력하면 폴더가 자동으로 만들어진다. 이 방법으로 파일 6개를 하나씩 새로 만들었다.

5. GitHub Desktop으로 갈아타다
근데 매번 파일 하나하나 웹에서 만드는 게 너무 번거로웠다. UI도 솔직히 나한텐 좀 불편했다. 그래서 GitHub Desktop이라는 프로그램을 깔았다.

설치하고 로그인까지는 됐는데, 처음엔 엉뚱한 튜토리얼용 저장소(desktop-tutorial)가 열려서 당황했다. Clone 화면에서도 이름이 비슷한 저장소가 여러 개 떠서 (github.io, kilimi84.github.io) 잘못 고를 뻔했다.

결국 진짜 저장소(kilimi84.github.io)를 정확히 골라서 컴퓨터로 내려받는 데 성공했다.

6. 그리고 지금
우여곡절 끝에 사이트가 제대로 떴다. 목록에 날짜와 글 제목이 뜨는 모습을 보니 은근 뿌듯했다.

돌아보면 별거 아닌 과정인데, 폴더 구조 하나 때문에 한참을 헤맸다. 근데 이런 삽질이 결국 다 기록이 되는 거 아닐까. 이 글 자체가 그 증거다.

오늘 배운 것

GitHub 웹 업로드는 폴더 구조를 못 알아먹는다
파일명에 슬래시 넣으면 폴더가 자동 생성된다
GitHub Desktop이 훨씬 편하다 (그리고 튜토리얼 저장소랑 헷갈리지 말 것)
